Design Mechanics · Predictor locked (between pick and resolution)
Live Stack pill updates — LAL's "0:47 to lock" timer becomes "🔒 Locked" (green). User has committed; still focused.
Lock card replaces YES/NO buttons — green-tinted card shows user's pick prominently: YES (big) + 2× multiplier chip + +100 IQ if correct payout preview.
Live tracker — lower half of the lock card shows the real-time stat being measured. Example: "LeBron has 0 pts in Q3 so far · 6:42 left." Updates as the game progresses. Turns green as prediction becomes likely; red as it becomes unlikely.
Splits stay visible — Sharp $ and Crowd % remain as information / validation. Can show how consensus shifted since user picked.
Friends ticker localizes — shows what friends on the same game picked. Kim also picked YES at 1.5× — social reinforcement.
User can swap focus — tapping 🧠 Trivia pill moves them into the trivia round countdown, their predictor stays locked and tracking in the background. On resolution, the LAL pill flashes with result (see next mockup: predictor-result).
