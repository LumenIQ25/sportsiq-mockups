Design Mechanics · Bar screen trivia
Visual language shared with mobile — same colors, typography, category pills, progress dots, timer treatment. Patrons glancing between phone and TV see the same brand.
Designed for 10-15 ft viewing — big question text (30px scaled for desktop preview; on a real 4K TV this reads clearly across a bar). 2×2 answer grid uses full-width tiles. Timer is 34px amber with glow.
Top bar anchors venue identity — "Kincade's Sports Pub" next to SportsIQ logo. 3× bar bonus badge (green) and LIVE indicator (red pulsing).
Left sidebar = bar-specific context — "Tonight at Kincade's" leaderboard (top 5 at this venue only, separate from national). QR code panel: scan → phone drops into this bar's session at 3× multiplier.
Question area = national synchronized content — identical question, answers, progress, and timer as every mobile player in the country. Bar screen stays in lock-step with the shared round.
Bottom strip tells the story — global player count (4,218 nationally) + venue count (42 here) reinforces "you're part of a national game AND your bar's local competition." On the right: teaser for what's next ("🏀 Predictor · LAL/GSW") so patrons know to stay.
Sync spec — the backend pushes the same question+timer state to bar screen and all mobile clients at once. Bar's WebSocket channel receives identical payload. When timer expires, reveal state triggers on both simultaneously.
