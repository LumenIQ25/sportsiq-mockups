Design Mechanics
No bottom nav during gameplay — immersive focus. No distractions.
Progress dots (10) — green filled = correct, red = wrong, purple glow = current.
Timer — amber fill depletes over ~15 sec per question; turns red at <5 sec.
Category pill — sport/category indicator, subtle but informative (answers "what topic").
Multipliers (per-question) — matches production code in sportsiq-frontend bundle. 3 pills above answers, labeled "Risk multiplier?" — 1.5× (green, safe) · 2× (gold, balanced — default selected) · 3× (red, risky). Each pill shows the IQ payout for that risk level (base×multiplier). On wrong answer, loss is the inverse of the win amount at that multiplier. Multiplier emits `trivia:multiplier` event on tap; locks when answer is selected.
Answer buttons — full-width, big tap targets, letter prefix in a circle, fits 4 buttons without scroll.
Live count — ticking national number reinforces "you're playing with thousands right now."
After-answer state (next mockup): selected answer highlights, others lock; at timer expire, correct answer glows green + wrong glows red; auto-advance to next Q after ~2 sec.
Exit button (×) — taps to confirm leaving round (would lose progress).
